
# 个人健康看板 (AI Health Dashboard) 设计文档

## 1. 项目愿景

本项目是一个集成了 **佳明 (Garmin CN)** 运动数据同步与 **AI 视觉食物识别** 的个人健康管理本地 Web 应用。

* **初期目标**：构建本地运行的 MVP，实现“摄入”与“消耗”的数据闭环。
* **核心逻辑**：通过 AI 自动计算热量差，无需手动输入繁琐的数值。
* **扩展性预留**：采用前后端分离架构，方便未来迁移至 Android 原生应用。

---

## 2. 技术栈 (本地开发版)

| 模块 | 推荐技术 | 说明 |
| --- | --- | --- |
| **后端** | **Python (FastAPI)** | 高性能异步框架，拥有成熟的佳明 (python-garminconnect) 和 AI SDK。 |
| **前端** | **Vue 3 + Tailwind CSS** | 响应式布局，使用 **Vant UI**（移动端风格组件库）方便以后迁移。 |
| **数据库** | **SQLite** | 无需安装，单个文件存储，与 Android 的 Room/SQLite 无缝衔接。 |
| **AI 识别** | **Gemini 2.0 Flash** | 强大的多模态视觉能力，识别食物种类并直接输出营养成分 JSON。 |
| **佳明接口** | `python-garminconnect` | 支持 `is_cn=True` 配置，适配国内服务器。 |

---

## 3. 核心功能模块

### 3.1 佳明数据同步 (Garmin CN Sync)

* **功能**：每日定时或手动触发，从 `garmin.cn` 拉取数据。
* **关键数据点**：
* `active_calories`: 今日活动消耗热量。
* `bmr`: 基础代谢（静息热量）。
* `steps`: 今日步数。
* `sleep`: 昨日睡眠时长（总时长、深睡、浅睡）。



### 3.2 AI 视觉识别 (AI Nutritionist)

* **工作流**：用户上传/拍摄照片 -> 后端转 Base64 -> 调用 Gemini API。
* **Prompt 约束**：
> "作为营养专家，请识别图中的食物。输出 JSON 格式：`{"name": "...", "kcal": 0, "protein": 0, "fat": 0, "carb": 0}`。若有多个食物请汇总。只输出 JSON。"


* **存储**：将识别结果存入 `food_logs` 表，支持用户手动微调。

## 4. 界面布局 (UI/UX)

应用采用 **侧边栏 (Sidebar)** 导航模式，包含以下 4 个功能模块切换：

1. **总览 (Overview)**：
* **热量仪表盘**：`当前可摄入 = 目标 - 已吃 + (活动消耗 + BMR)`。
* **今日数据卡片**：步数进度条、睡眠时长（昨日）。


2. **无氧消耗 (Anaerobic)**：
* 展示 Garmin 记录的“力量训练”、“HIIT”等高强度数据。
* 展示心率区间分布。


3. **有氧消耗 (Aerobic)**：
* 展示跑步、步行、骑行等里程数据。
* 展示有氧运动消耗的热量趋势图。


4. **摄入 (Intake)**：
* **拍照/上传按钮**：核心交互入口。
* **今日记录列表**：显示每餐照片缩略图及识别到的热量明细。

## 5. 数据库设计 (SQLite)

### `user_config` (用户设置)

* `id` (PK), `target_kcal` (INT), `garmin_user`, `garmin_pass`.

### `daily_metrics` (每日活动汇总)

* `date` (TEXT, PK), `active_kcal`, `resting_kcal`, `steps`, `sleep_minutes`.

### `food_logs` (摄入记录)

* `id` (PK), `date`, `food_name`, `kcal`, `macros` (JSON 存储蛋白质/脂肪/碳水), `img_path`.
